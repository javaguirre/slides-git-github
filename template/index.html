Control de versiones

Gestión y actualización de versiones de nuestro proyecto

Repositorio: Archivo de versiones de nuestro proyecto

Versiones: Distintas actualizaciones de un proyecto que se archivan en el repositorio.

Una versión nueva es un commit

Centralizado/distribuido

  • Repositorio principal
  • Ramas
  • Conflictos
  • Colaboración sencilla

Funcionamiento

  • Control de ficheros
  • Guardar nueva versión
  • Sincronizacion de código
  • Corrección de errores
  • Despliegue del proyecto

Introducción/Historia

  • Creado por Linus Torvalds
  • Control de versiones distribuido
  • Estándar de facto en el mundo del SL

Creación

Creación del proyecto

cd miproyecto
git init

Copia de un proyecto

git clone url_proyecto

Primera versión

touch README  # añadimos algo de información
git add README
git commit -m 'Primera version'

Manejo de archivos

Manejando los archivos sometidos a versiones

git add archivonuevo
git rm archivoviejo
git mv archivoviejo archivonuevo
Fuente:

Log

¿Qué hay en nuestro proyecto?

git log  # Lista los últimos commits
git log -n 30  # 30 ultimos commits
git log -p  # ultimos cambios en codigo
git log --stat  # ultimos cambios en archivos

Revertir cambios

Volvemos atrás en nuestro código

git checkout [HASH] archivos
git reset --hard SHA  # no hemos creado una nueva versión
git revert HEAD  # crea un nuevo commit

Sincronización remota 

#Actualiza copia local
git pull repo_remoto rama

#Actualiza copia remota
git push repo_remoto rama

#Actualizando objetos de la copia local
git fetch
git merge repo_remoto rama

Parcheando

# Creamos un parche con la diferencia
git diff archivos > miarchivo.patch
# Aplicamos el parche en nuestro repo
git apply miarchivo.patch

Ramas - Funcionalidad

Creamos una nueva funcionalidad en nuestro proyecto

git branch mi_nueva_funcionalidad
git checkout mi_nueva_funcionalidad

Cuando acabemos podemos unirla a principal

# Volvemos al proyecto principal
git checkout master
# Nos traemos los cambios de la
# nueva funcionalidad estable
git merge mi_nueva_funcionalidad
# Conflicto!
# o
# Si todo va bien...
git push origin master

Conflictos!

git checkout --ours filename.c
git checkout --theirs filename.c

git show :1:file.txt  # la unión de ambas
git show :2:file.txt  # mi version.
git show :3:file.txt  # la versión obtenida

# Finalmente..
git add archivos
git commit
git push origin master
